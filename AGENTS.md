# AGENTS.md — MeaPet 桌宠

## Project overview
Desktop pet app (PyQt5 transparent window) with AI chat, screen watching, TTS voice, and Live2D/PNG sprite rendering. Windows-first, Linux supported.

## Entry point & config
- **Entry**: `python pet.py` (Linux: `QT_QPA_PLATFORM=xcb python pet.py`)
- **Setup**: `python setup_wizard.py` (GUI) or copy `config.example.json` → `config.json`
- `config.json` is **gitignored**; bubble durations live in `config.json` (`bubble_duration_ms`); example in `config.example.json`
- Python version: **3.10–3.12** (`.python-version` = 3.10.20). 3.12+ needs `numpy<2` and `setuptools==69.5.1`

## Architecture

| Path | Role |
|------|------|
| `pet.py` / `meapet/desktop/app.py` | Thin entry + desktop pet (mixins: chat/watch/render/audio) |
| `meapet/chat/engine.py` | LLM engine (Ollama / DeepSeek / MiMo), async httpx |
| `meapet/memory/db.py` | SQLite memory + affection (`mea_memory.db`, RLock) |
| `meapet/watcher/screen.py` | Screen watch QThread + privacy gates |
| `meapet/tts/` | GSV / VITS / MiMo TTS engines |
| `wizard/` | Setup wizard pages |
| `meapet/utils.py` | `safe_print`, secret redaction, watcher helpers |

## Critical gotchas

### Import order: `socket` before PyQt5
`pet.py:17` and `chat.py:8` import `socket` before any PyQt5 import to avoid QtNetwork conflicts. Do not reorganize imports.

### `ensure_utf8_stdout()` is called **once** in `pet.py:31`
Other modules must NOT wrap stdout again — the wrapper is shared; re-wrapping would close the underlying buffer when the old wrapper is GC'd.

### Threading model (no QThread for chat/TTS)
- `ChatWorker` and `TTSWorker` are bare `threading.Thread` wrappers (NOT QThread)
- `ScreenWatcher` is a proper `QThread` subclass
- Main thread polls workers via `QTimer` every 100ms; do not introduce blocking calls in main event loop

### Config files are separate
- `config.json` — user-editable (LLM backend, TTS, character, display). gitignored.
- `weight.json` — TTS model weight registry. Tracked.

### No tests, no linter, no CI
This is a personal project. There are no test suites, formatters, or CI workflows. The only verification is manual run.

## Key behaviors
- **Chat history**: max 8 messages (system prompt + 6 user/assistant pairs), trimmed in `chat.py:142-144`
- **Memory extraction**: triggers every 3 user messages or when user says "记住"/"记下"/etc. Uses Ollama `/api/generate`
- **Affection**: range 90–100, daily cap of 15, starts at 90
- **Screen watching**: timer fires every 3–6 min randomly. Suppressed if user interacted <3 min ago or standby mode is on
- **Voice cache**: `voice_cache/{jp|zh}_{safe_name}.wav`. Pre-generated by `precache_interactions.py`
- **Bubble durations**: controlled by `config_settings.json` per category (default/reply/watch/interaction/thinking)
