## Project overview
Desktop pet app (PyQt5 transparent window) with AI chat, screen watching, TTS voice, and Live2D/PNG sprite rendering. Windows-first, Linux supported.

## Entry point & config
- **Entry**: `python pet.py` (Linux: `QT_QPA_PLATFORM=xcb python pet.py`)
- **Setup**: `python setup_wizard.py` (GUI) or copy `config.example.json` ‚Üí `config.json`
- `config.json` is **gitignored**; `config_settings.json` (bubble durations, TTS sync) is tracked
- Python version: **3.10‚Äì3.12** (`.python-version` = 3.10.20). 3.12+ needs `numpy<2` and `setuptools==69.5.1`

## Architecture

| File | Role |
|------|------|
| `pet.py` | Main app (~1880 lines). MeaPet QWidget, all UI, worker threads, event wiring |
| `chat.py` | LLM engine: Ollama or DeepSeek API. History ‚â§8 msgs. Memory extraction every 3 turns |
| `memory.py` | SQLite (`mea_memory.db`): chat history, affection (90-100 scale), memories, events |
| `watcher.py` | ScreenWatch QThread: 3-tier (capture‚Üísummary‚Üídecision‚Üíreply). Ollama vision models |
| `tts.py` | GPT-SoVITS TTS via subprocess. Auto-translates CN‚ÜíJP, caches to `voice_cache/` |

| `utils.py` | `safe_print` (stderr), `log_error` (file), `ensure_utf8_stdout` (Windows) |

## Critical gotchas

### Import order: `socket` before PyQt5
`pet.py:17` and `chat.py:8` import `socket` before any PyQt5 import to avoid QtNetwork conflicts. Do not reorganize imports.

### `ensure_utf8_stdout()` is called **once** in `pet.py:31`
Other modules must NOT wrap stdout again ‚Äî the wrapper is shared; re-wrapping would close the underlying buffer when the old wrapper is GC'd.

### Threading model (no QThread for chat/TTS)
- `ChatWorker` and `TTSWorker` are bare `threading.Thread` wrappers (NOT QThread)
- `ScreenWatcher` is a proper `QThread` subclass
- Main thread polls workers via `QTimer` every 100ms; do not introduce blocking calls in main event loop

### Config files are separate
- `config.json` ‚Äî user-editable (LLM backend, TTS, character, display). gitignored.
- `config_settings.json` ‚Äî bubble durations per category, TTS sync flag. Tracked in git.
- `weight.json` ‚Äî TTS model weight registry. Tracked.

### No tests, no linter, no CI
This is a personal project. There are no test suites, formatters, or CI workflows. The only verification is manual run.

## Key behaviors
- **Chat history**: max 8 messages (system prompt + 6 user/assistant pairs), trimmed in `chat.py:142-144`
- **Memory extraction**: triggers every 3 user messages or when user says "ËÆ∞‰Ωè"/"ËÆ∞‰∏ã"/etc. Uses Ollama `/api/generate`
- **Affection**: range 90‚Äì100, daily cap of 15, starts at 90
- **Screen watching**: timer fires every 3‚Äì6 min randomly. Suppressed if user interacted <3 min ago or standby mode is on
- **Voice cache**: `voice_cache/{jp|zh}_{safe_name}.wav`. Pre-generated by `precache_interactions.py`
- **Bubble durations**: controlled by `config_settings.json` per category (default/reply/watch/interaction/thinking)
